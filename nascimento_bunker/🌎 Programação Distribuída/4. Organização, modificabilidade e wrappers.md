## Organização
Refere-se a como um sistema deve ser estruturado.
**Exemplos** 
- Arquitetura de camadas, bem organizadas por responsabilidades
- Microserviços, organizados por serviços.
Claro com responsabilidades bem divididas e de fácil compreensão e navegação.

## Modificabilidade
Fácil de alterar, evoluir e estender.
**Exemplos** 
- Bem desacoplado são fáceis de modificar
- Monolíticos são difíceis

## Wrappers
Embrulham um sistema ou componente, escondendo sua complexidade
**Exemplo**
- Criar uma API moderna para um Legado
- Encapsular acesso à um banco de dados

### Interceptadores
São componentes que captura chamadas antes e depois de acontecerem.
**Exemplo** 
- Registradores de Logs
- Autenticadores 
- Controladores de transação

Esses três conceitos trabalham juntos:

- 🧱 Boa organização → sistema compreensível
- 🔧 Alta modificabilidade → sistema evolui fácil
- 🧩 Wrappers → isolam complexidade
- 🎭 Interceptadores → adicionam comportamento sem alterar código
